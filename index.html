Build Loop

从灵感到验收，
每一步都留下证据

我会把需求、页面、接口、测试截图和说明文档串起来，让项目不仅能运行，也能在面试里被快速理解。

需求拆解

定义用户场景、核心流程、数据结构和最小可用功能，避免只做视觉壳。

02 / Build

工程实现

按页面、接口、工具层和资源层拆分，实现可运行、可调试的版本。

03 / Verify

测试验证

用脚本、单元测试、语法检查和浏览器截图验证核心逻辑与页面质量。

04 / Present

作品表达

整理项目展示页、截图、视频、技术说明和简历要点，让项目易于被理解。
